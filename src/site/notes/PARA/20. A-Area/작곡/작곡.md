# 프로세스

작곡은 대충 이런 식으로

- [[PARA/20. A-Area/작곡/송폼\|송폼]] 구상하기 : 곡이 전체적으로 길이가 얼마나 될지, 곡의 구조 등을 생각한다.
- 곡의 틀 짜기: 전체적 느낌, 곡의 [[PARA/20. A-Area/작곡/리듬\|리듬]], 코드 톤, 드럼 패턴 등을 추가한다.
	- 코드 : [[머니코드\|머니코드]] 나 [[2-5-1\|2-5-1]] 참조
	- 드럼 패턴 : [[PARA/20. A-Area/작곡/드럼 패턴\|드럼 패턴]] 참조.
- 탑라인 멜로디 : 보컬이나 전체적 곡의 멜로디를 추가한다.
- 여러 이펙트 (e.g. [[PARA/20. A-Area/작곡/Reverb\|Reverb]] 등) 나 FX (e.g. Riser, Sweeper 등) 을 추가한다
	- [[PARA/20. A-Area/작곡/샘플/샘플\|샘플]] : Fx는 왠만하면 기존에 있는 샘플을 사용하는 게 편하다.
	- [[PARA/20. A-Area/작곡/샘플/샘플\|샘플]] : 리믹스를 하거나 기존의 곡 일부를 샘플링 할 때 쓸만한 사이트와 정보들에 대해서도 기술한다.
- 믹싱 : 각 악기들의 소리가 적절한지, 기타 상세를 조정한다

# 작곡 처음이에요! 뭐부터 하나요?
- 코드, 스케일, 모드(선법), 음정 등 화성학적 지식 공부하기
- 많이 듣기 : 좋아하는 장르, 아티스트, 곡은 무엇인가? 왜 좋은가? 비슷한 걸 만든다면? 똑같이 하려면 어떻게 할까? 다르게 한다면 어딜 어떻다르게 할까?
- DAW 공부하기 : [[PARA/20. A-Area/작곡/Reaper/Reaper\|Reaper]] 나 Ableton, Logic, cubase 등 다양한 종류의 DAW가 있다.
- 플러그인 공부하기 : 각종 FX, 예를들어 필터, [[PARA/20. A-Area/작곡/ADSR Envelope\|ADSR Envelope]], [[PARA/20. A-Area/작곡/Reverb\|Reverb]], [[Delay\|Delay]] 등, 또는 사용되는 VST (가상악기) 등에 대해서 공부한다.
- 기존 곡 분석 및 카피 : 곡의 구조나 BPM 등 [[PARA/20. A-Area/작곡/송폼\|송폼]] 에 대한 것, 사용되는 가상악기의 음색, 사용된 코드 톤, 멜로디 전개 방식 등을 분석하다보면 내가 들었을때 어떤 점이 좋았다는 것을, 나중에 창작할 때 써먹을 수 있다.
- 리듬에 대한 분석 : 드럼이나 베이스, 멜로디의 리듬과 패턴. 강박/약박에 대한 이해 등. 같은 곡을 다른 리듬으로 연주해보기 등.

## [linux](https://imaginative-griffin-caaf97.netlify.app/404) 기본 세팅

[How to install and use VST Plugins on Linux](https://linuxmusician.com/how-to-install-and-use-vst-plugins-on-linux/)  
VST plugins come in two versions:

- **VST2**: Older format, widely supported.
- **VST3**: Newer format with improved performance and features.  
    Linux supports both formats, but VST2 has broader compatibility with Linux-based DAWs (Digital Audio Workstations).
- **Update your system:** `sudo apt update && sudo apt upgrade`
- **Install essential audio libraries:** `sudo apt install wine-stable winetricks jackd qjackctl`
- **Install your preferred DAW:** Popular options include Ardour, Bitwig, Reaper, and LMMS.

Native VST plugins are designed to work directly with Linux systems.

1. **Download the plugin:** Visit the developer’s site and download the Linux version.
2. **Extract files:** If the file is compressed (.tar.gz, .zip), extract it: `tar -xvzf plugin-name.tar.gz`
3. **Move the plugin to the VST folder:** `sudo mv plugin.so /usr/lib/vst/`
4. **Refresh your DAW’s plugin list:** Open your DAW and scan for new plugins.
